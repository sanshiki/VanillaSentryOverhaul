## 🎯 总目标

将项目中所有可能导致多人不同步的问题修改为：

* 服务器权威逻辑
* 正确的 owner 判断
* 正确的 netUpdate 使用
* 不破坏原有单人逻辑

---

# 一、实体生成规则

---

## 规则 1：在 AI / PostAI / OnHit / Update 等函数中生成实体时

### 🔍 检索：

搜索：

NewProjectile
NewProjectileDirect
NewNPC
NewItem

---

### 📌 判断所在函数：

如果在以下函数中：

* AI()
* PostAI()
* OnHitNPC()
* OnHitPlayer()
* OnTileCollide()
* ModPlayer.PostUpdate()
* GlobalProjectile.*
* NPC.AI()

---

### 🔧 修改规则：

### 情况 A：当前类是玩家拥有的 Projectile（有 owner）

将：

```
Projectile.NewProjectile
```

改为：

```
(Projectile.owner == Main.myPlayer)
{
    Projectile.NewProjectile
}
```

**注意：只给关键的生成实体那一段代码添加包裹！！不要给其他逻辑添加！！**

---

### 情况 B：非玩家拥有实体（NPC / 系统逻辑）

改为：

if(Main.netMode != NetmodeID.MultiplayerClient)
{
    // 原生成代码
}

---

❗ 不允许两种方式混用。

# 四、localAI 规则

---

## 规则 4：禁止将重要逻辑放在 localAI

localAI 只允许用于：

* 动画
* 本地计时
* 粒子效果

禁止用于：

* 伤害逻辑
* 攻击间隔控制（会不同步）
* 状态机主状态

如果发现 localAI 控制攻击行为：

→ 改为 ai[]

# 十、禁止修改内容

---

Cursor 不得：

* 修改 return false / true 行为
* 修改 owner 参数
* 删除 netUpdate
* 修改 Projectile.owner
* 删除 ai[] 数组

除非明确指示。

---

# 十一、多人安全判断优先级

Cursor 在遇到生成逻辑时，按以下顺序判断：

1. 是否在 Item.Shoot → 不修改
2. 是否为玩家拥有 projectile → 用 owner 判断
3. 是否为 NPC 或系统 → 用 netMode 判断
4. 是否为 GlobalProjectile → 必须服务器执行

---

# 十二、修改后必须保证

* 单人模式行为不变
* 多人不会重复生成
* 不出现 rubberband
* 不出现重复伤害
